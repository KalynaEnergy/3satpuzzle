How to read it. Each column is a switch; each row is a rule. A rule is a clause of (up to) three demands joined by OR, so a row is satisfied the moment any one of its cells is green. Because demands sit under their switch, you can scan a column to see every rule that switch touches — and predict exactly which cells a flip would turn from red to green or back.

How it maps to 3‑SAT. Switches are boolean variables, a demand like ON/OFF is a literal, a row is a 3-literal clause, and the board is those clauses joined by AND. Every puzzle has exactly one solution: a solution is planted, then rules are added — each aimed at a wrong answer still standing — until only the intended arrangement survives. Pure deduction, no guessing required.
